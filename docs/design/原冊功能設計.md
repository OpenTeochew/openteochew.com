# 原冊功能設計

## 目標

搜索結果和詞條詳情中的每個詞條有一個「原冊」按鈕，點擊跳到 SourceViewer 頁面，定位到該詞條在原書的對應頁面圖片。

## 背景

- Source 001（Handbook of the Swatow Vernacular）在維基文庫有完整 OCR
- 本地有 PDF（304 頁，`/Users/lim/Documents/hw-onedrive/book/Handbook_of_the_Swatow_Vernacular.pdf`）
- 維基文庫使用子頁面結構（如 `Handbook_of_the_Swatow_Vernacular/Relationships`）
- 子頁面 HTML 中有 `data-page-index` 屬性，可提取 PDF 頁碼
- CSV 數據中大部分 entries 的 `page_num` 為 NULL

## 設計

### A. 數據管道（可重複執行，支持增量同步）

#### 1. PDF 分割腳本：`scripts/split-pdf.mjs`

- 輸入：本地 PDF 文件路徑
- 輸出：`backend/public/scans/{source_id}/` 下的單頁 PNG
- 使用 `ghostscript`（`gs`）或 `pdftoppm`
- 冪等：已存在的頁可跳過

#### 2. 維基文庫同步腳本：`scripts/sync-wikisource.mjs`

從維基文庫抓取頁碼映射和 OCR 文字：

1. 抓取主頁面 HTML，解析所有子頁面連結
2. 抓取每個子頁面 HTML，提取：
   - `data-page-index`（djvu/PDF 頁碼）
   - 章節標題（`<h2>` / `<h3>` 的 id）
   - OCR 文字內容（清理 HTML 標籤）
3. 建立 section_title → page_num 的映射
4. 生成 SQL：
   - `UPDATE entries SET page_num = ? WHERE source_id = ? AND section_id IN (SELECT id FROM sections WHERE title = ?)`
   - `INSERT OR REPLACE INTO pages (section_id, page_num, image_url, ocr_text, sort_order)`

映射邏輯：
- CSV 的 section title 格式：`Lesson XXV. relationships.`
- 維基文庫子頁面標題：`Relationships`
- 需要一個從 section title 到 wikisource subpage 的映射表（硬編碼在腳本中，或存在 DB）

### B. 後端

- 現有 `/api/v1/sources/:id/pages` 和 `/api/v1/sources/:id/entries` 已支持 `page_num` 查詢
- `pages.image_url` 指向 `/scans/001/NNN.png`（靜態文件由 Workers 提供）
- `sources` 表加 `pdf_url` 或 `wikisource_url` 欄位（可選，記錄來源）

### C. 前端

#### SearchResults
- 每行 entry 加「原冊」圖標按鈕
- 僅當 `entry.page_num` 存在時顯示
- 連結：`/thak/source/:source_id?page=PAGE_NUM`

#### EntryDetail
- 在 def-block 的 source 行加「原冊」連結
- 連結同上

#### SourceViewer
- 接收 `?page=XX` query 參數，初始化時定位到對應頁
- 顯示本地圖片（`/scans/001/NNN.png`）
- 側邊欄顯示該頁詞條列表（從 API 查 `entries WHERE page_num = ?`）

## 實現順序

1. PDF 分割 → 生成頁圖片
2. 維基文庫同步腳本 → 填充 entries.page_num 和 pages 表
3. SourceViewer 支持圖片顯示 + page query 參數
4. SearchResults / EntryDetail 加「原冊」按鈕

## 維基文庫同步策略

- 腳本可重複執行（冪等）
- 維基文庫有更新時，重新執行即可同步 page_num 和 OCR
- 未來可整合到 CI workflow
